# design-sync notes — kd-design-system

- This repo is CSS-first: the source of truth is `css/tokens.css` + `css/kd.css`. The React layer (`src/index.tsx`) is thin wrappers applying those classes — created 2026-07-22 for the sync. **If kd.css class names change, update the wrappers.**
- Build: `npm run build` — esbuild (dist/index.js) + tsc d.ts + `cat css/tokens.css css/kd.css > dist/kd.css`. `cfg.cssEntry` points at `dist/kd.css`, so ALWAYS rebuild before the converter or shipped tokens go stale.
- `tokensGlob` does not work here (it requires a tokens package in node_modules); tokens ride into `_ds_bundle.css` via the concatenated `cssEntry`. Don't re-add it.
- Fonts: Instrument Sans variable woff2s vendored in `fonts/` (downloaded from Google Fonts, SIL OFL); `fonts/instrument-sans.css` wired via `cfg.extraFonts`. The doc site itself still loads from Google Fonts — unrelated.
- `KdProvider` is mandatory context (typography scope + theme tokens) — `cfg.provider` is set to it; previews render unstyled text without it.
- `dist/` and `node_modules/` are gitignored: on a fresh clone run `npm install && npm run build` before the converter.
- Playwright: chromium-headless-shell v1228 cached (installed 2026-07-22 via `.ds-sync`'s playwright).
- Known render warns: none outstanding (the Stat/ThemeToggle `[RENDER_THIN]` warns were pre-authoring floor cards, resolved by authored previews).

## Re-sync risks

- Preview content name-drops real portfolio clients (Mistara, ASL Airlines, Liffey Meats, Engineers Ireland) — deliberate, matches the portfolio; keep if those case studies stay public.
- The `Heading` component's `role` prop shadows the ARIA `role` attribute in its props type — intentional API choice; don't "fix" it without updating previews and docs.
- `dist/kd.css` is generated by concatenation; if kd.css ever gains its own `@import`s, the concat approach breaks (imports must come first in CSS) — switch to a real CSS build then.
- The doc site (index.html/css/js at repo root) is independent of the sync; changes there don't require a re-sync unless tokens/kd.css changed.
